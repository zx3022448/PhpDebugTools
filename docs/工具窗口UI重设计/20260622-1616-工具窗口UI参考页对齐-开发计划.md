# 工具窗口 UI 参考页对齐 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变现有方法直调逻辑的前提下，将工具窗口 UI 重组为接近参考 HTML 的 Apple 风格调试工作台。

**Architecture:** 以 `PhpDebugToolsToolWindowPanel` 提供外层拟态窗口壳，以 `MethodInvokeToolWindowPanel` 提供左侧请求工作台与右侧状态结果侧栏。保留现有业务组件与执行链路，仅重做容器结构、样式系统和少量状态展示映射。

**Tech Stack:** Kotlin、Swing、IntelliJ Platform UI 组件、JUnit 4、Gradle

---

## 文件结构与职责映射

### 主要修改文件

- `src/main/kotlin/com/example/phpdebugtools/toolwindow/ToolWindowUiStyles.kt`
  - 扩展为统一的参考页风格样式中心，提供窗口壳、标题栏、品牌卡、导航 chips、最近执行卡、浅色卡片、深色代码面板、指标卡和状态卡样式。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/PhpDebugToolsToolWindowPanel.kt`
  - 将工具窗口重组为外层工作台壳，增加假标题栏、品牌导航区、最近执行区，并承载现有 `概览` 与 `方法直调` 入口。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/MethodInvokeToolWindowPanel.kt`
  - 将方法直调页改成左侧请求工作台加右侧状态结果栏，复用现有输入组件和执行逻辑。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/ControllerRequestEditorPanel.kt`
  - 将请求方式、Query、Body、Headers 和 Body 子模式切换调整为胶囊式和卡片式排版。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/RequestParameterTablePanel.kt`
  - 调整参数表格工具条、表头、滚动区和默认容器样式，以适配参考页的浅色工作台。
- `src/main/kotlin/com/example/phpdebugtools/ui/ServiceMethodInputPanel.kt`
  - 调整服务参数编辑器容器风格，使其与新的浅色工作台卡片保持一致。
- `src/main/resources/messages/MyMessageBundle.properties`
  - 补充标题栏、导航、状态卡、指标卡、最近执行和空状态相关文案。
- `src/test/kotlin/com/example/phpdebugtools/toolwindow/MethodInvokeToolWindowPanelTest.kt`
  - 调整并补充关键结构断言，确保新布局仍保留核心交互与组件挂载。
- `src/test/kotlin/com/example/phpdebugtools/toolwindow/RequestParameterTablePanelTest.kt`
  - 保持参数表格最小行为测试，并在需要时补充样式承载结构的探针验证。

- [ ] **Step 2: 左侧请求工作台保留现有搜索、方法选择、PHP 选择和执行按钮**

1. `searchField` 放入顶部操作行

3. `phpExecutableComboBox` 放入顶部操作行或次级操作位
4. `executeButton` 保持高强调按钮
5. `refreshButton`、`phpRefreshButton` 改为圆形或轻量按钮

- [ ] **Step 3: 将现有 top tabs 和 cards 调整为胶囊式 section tabs**

```kotlin

MainContentCard.SCRIPT
```

允许将 `ControllerRequestEditorPanel` 和参数编辑区拆到更贴近参考页的位置，但不要改动其业务方法签名。

- [ ] **Step 4: 将右侧状态与结果区重组为状态卡、指标卡和深色输出面板**

指标卡优先展示：

2. 运行时选择摘要
