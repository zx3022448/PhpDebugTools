# 工具窗口 UI 重设计 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改变现有方法直调功能和运行时协议的前提下，将工具窗口重组为跟随 IDE 主题的调试工作台界面，并补齐最小可维护的布局验证。

**Architecture:** 通过为工具窗口引入轻量 UI 样式支撑层，把现有 `MethodInvokeToolWindowPanel`、`ControllerRequestEditorPanel` 和 `RequestParameterTablePanel` 从传统表单布局重组为卡片式工作台。保留现有方法搜索、请求组装、PHP 运行时选择和执行链路，仅替换组件拼装方式、视觉层级和少量文案结构。

**Tech Stack:** Kotlin、Swing、IntelliJ Platform UI 组件、JUnit 4、IntelliJ Platform Test Framework

---

## 文件结构与职责映射

### 现有文件

- `src/main/kotlin/com/example/phpdebugtools/toolwindow/MethodInvokeToolWindowPanel.kt`
  负责方法直调主页面；本次会重组为顶部操作头部区、方法信息卡片、参数编辑卡片、请求上下文卡片和底部结果区。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/ControllerRequestEditorPanel.kt`
  负责控制器方法的请求上下文编辑；本次主要调整请求方式行、标签页和 Body 模式切换区的排版与样式。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/RequestParameterTablePanel.kt`
  负责 Query/Header/Body 参数表格；本次主要调整工具条、表格风格、滚动容器和行高列宽。
- `src/main/kotlin/com/example/phpdebugtools/toolwindow/PhpDebugToolsToolWindowPanel.kt`
  负责工具窗口整体结构；本次只做概览页最小视觉统一。
- `src/main/kotlin/com/example/phpdebugtools/ui/ServiceMethodInputPanel.kt`
  负责服务方法参数编辑；本次只做容器层级适配，不重写参数采集逻辑。
- `src/main/resources/messages/MyMessageBundle.properties`
  负责工具窗口文案；本次会补齐卡片标题、状态标签和更短的提示语。

### 新增文件

- `src/main/kotlin/com/example/phpdebugtools/toolwindow/ToolWindowUiStyles.kt`
  集中封装颜色读取、卡片样式、边框、内边距、标签状态、主次按钮、滚动区与文本区样式。
- `src/test/kotlin/com/example/phpdebugtools/toolwindow/MethodInvokeToolWindowPanelTest.kt`
  覆盖方法直调页重组后仍需保持的关键结构，如顶部操作区存在、结果区默认可见、服务参数表单仍可切换。
- `src/test/kotlin/com/example/phpdebugtools/toolwindow/RequestParameterTablePanelTest.kt`
  覆盖参数表格的最小结构行为，如默认行、添加删除动作、表头列名与样式入口存在。

### 不拆分的边界说明

- 不新增独立的“主题管理器”或复杂自定义绘制类，只新增一个轻量 `ToolWindowUiStyles.kt`。
- 不拆分 `MethodInvokeToolWindowPanel.kt` 为多个大文件，避免本轮为样式重构引入过度架构调整。

package com.example.phpdebugtools.toolwindow

import com.intellij.openapi.util.SystemInfoRt
import com.intellij.ui.JBColor
import com.intellij.util.ui.JBUI
import java.awt.Color
import javax.swing.AbstractButton
import javax.swing.BorderFactory
import javax.swing.JComponent
import javax.swing.JScrollPane
import javax.swing.JTextArea

internal object ToolWindowUiStyles {
    private val panelLight = JBColor(Color(0xF7F8FA), Color(0x242832))
    private val cardLight = JBColor(Color(0xFFFFFF), Color(0x2B3140))
    private val innerLight = JBColor(Color(0xF4F6FA), Color(0x20252E))
    private val borderLight = JBColor(Color(0xD9DEE7), Color(0x3A4250))
    private val successLight = JBColor(Color(0x0F9D58), Color(0x55D88A))
    private val warningLight = JBColor(Color(0xC47F00), Color(0xFFCC66))
    private val dangerLight = JBColor(Color(0xD93025), Color(0xFF7B72))

    fun applyWorkbenchSurface(component: JComponent) {
        component.isOpaque = true
        component.background = panelLight
        component.border = JBUI.Borders.empty(12)
    }

    fun applyCard(component: JComponent) {
        component.isOpaque = true
        component.background = cardLight
        component.border = JBUI.Borders.compound(
            BorderFactory.createLineBorder(borderLight, 1, true),
            JBUI.Borders.empty(12),
        )
    }

    fun applyInnerSurface(component: JComponent) {
        component.isOpaque = true
        component.background = innerLight
        component.border = JBUI.Borders.compound(
            BorderFactory.createLineBorder(borderLight, 1, true),
            JBUI.Borders.empty(8),
        )
    }

    fun applyPrimaryButton(button: AbstractButton) {
        button.background = JBColor.namedColor("Button.default.startBackground", warningLight)
        button.foreground = JBColor.namedColor("Button.default.foreground", JBColor.WHITE)
        button.border = JBUI.Borders.empty(6, 12)
        button.isFocusPainted = false
    }

    fun applySecondaryButton(button: AbstractButton) {
        button.background = innerLight
        button.foreground = JBColor.foreground()
        button.border = JBUI.Borders.compound(
            BorderFactory.createLineBorder(borderLight, 1, true),
            JBUI.Borders.empty(6, 10),
        )
        button.isFocusPainted = false
    }

    fun applyInputSurface(component: JComponent) {
        component.background = innerLight
        component.border = JBUI.Borders.compound(
            BorderFactory.createLineBorder(borderLight, 1, true),
            JBUI.Borders.empty(6, 8),
        )
    }

    fun applyResultArea(area: JTextArea) {
        area.background = innerLight
        area.border = JBUI.Borders.empty(8)
    }

    fun applyScrollPane(scrollPane: JScrollPane) {
        scrollPane.border = JBUI.Borders.empty()
        scrollPane.viewport.background = scrollPane.background
        if (SystemInfoRt.isMac) {
            scrollPane.putClientProperty("JScrollPane.style", "overlay")
        }
    }

    fun statusColor(status: MethodInvokeVisualStatus): Color = when (status) {
        MethodInvokeVisualStatus.IDLE -> JBColor.namedColor("Component.infoForeground", JBColor.GRAY)
        MethodInvokeVisualStatus.RUNNING -> warningLight
        MethodInvokeVisualStatus.SUCCESS -> successLight
        MethodInvokeVisualStatus.ERROR -> dangerLight
    }
}

internal enum class MethodInvokeVisualStatus {
    IDLE,
    RUNNING,
    SUCCESS,
    ERROR,
}

private fun buildWorkbenchContent(): JComponent {
    val container = JBPanel<JBPanel<*>>()
    container.layout = BoxLayout(container, BoxLayout.Y_AXIS)
    ToolWindowUiStyles.applyWorkbenchSurface(container)

    container.add(buildOperationCard())
    container.add(Box.createVerticalStrut(12))
    container.add(buildSummaryCard())
    container.add(Box.createVerticalStrut(12))
    container.add(buildArgsCard())
    container.add(Box.createVerticalStrut(12))
    container.add(buildResultCard())

    return container
}

internal fun hasWorkbenchSectionTitles(): Boolean =
    listOf(
        PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.operation"),
        PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.summary"),
        PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.result"),
    ).all(::allSectionTitlesContains)

internal fun hasSummaryAndResultSections(): Boolean =
    hasWorkbenchSectionTitles()

private fun allSectionTitlesContains(expected: String): Boolean {
    val queue = ArrayDeque<java.awt.Component>()
    queue.add(this)
    while (queue.isNotEmpty()) {
        val component = queue.removeFirst()
        if (component is JBLabel && component.text == expected) {
            return true
        }
        if (component is java.awt.Container) {
            component.components.forEach(queue::addLast)
        }
    }
    return false
}

private fun buildOperationCard(): JComponent =
    buildCard(PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.operation")) {
        add(buildOperationRows(), BorderLayout.CENTER)
    }

private fun buildSummaryCard(): JComponent =
    buildCard(PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.summary")) {
        add(buildSummaryContent(), BorderLayout.CENTER)
    }

private fun buildArgsCard(): JComponent =
    buildCard(PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.args")) {
        add(buildArgsContent(), BorderLayout.CENTER)
    }

private fun buildResultCard(): JComponent =
    buildCard(PhpDebugToolsBundle.message("toolwindow.methodInvoke.section.result")) {
