# 工具窗口 UI 重设计需求文档

## 1. 背景

当前 `PHP Debug Tools` 工具窗口已经具备方法搜索、PHP 运行时选择、服务方法参数输入、控制器请求上下文编辑与执行结果展示等核心能力，但整体界面仍以传统表单式布局为主，信息密度、视觉层次与主操作链路的连贯性不足。

本次重设计以参考图中的 API 调试工作台为视觉方向，目标是在**不改变现有功能结构和数据流**的前提下，重组方法直调页面的布局与样式，使其在 JetBrains IDE 中更像一个高频使用的调试工作台，而不是单纯的配置表单页。

## 2. 目标

本次 UI 重设计需要同时满足以下目标：

1. 保留现有功能结构与执行逻辑，不改动方法调用协议与运行时协议。
2. 跟随 IDE 主题切换，在深色主题下尽量贴近参考图气质。
3. 优先优化布局层次与信息密度，让主链路更清晰。
4. 将高频操作集中到顶部，减少滚动与视觉跳转。
5. 让参数编辑、请求上下文和执行结果具备更明确的卡片化分区。

## 3. 非目标

本次改造明确不包含以下内容：

1. 不新增新的功能 Tab。
2. 不修改方法执行逻辑、请求组装逻辑或运行时脚本。
3. 不引入第三方 UI 库。
4. 不做完全固定的网页式深色皮肤。
5. 不将整个插件所有页面一次性重绘，只优先优化方法直调页。

## 4. 设计范围

### 4.1 本次重点页面

重点改造 `方法直调` 页面。

### 4.2 低优先级统一项

`概览` 页只做最低限度的视觉统一，不做完整结构重塑。

## 5. 工作台布局设计

### 5.1 顶部操作头部区

顶部区域改造成两行高信息密度操作条：

1. 第一行放置方法搜索、候选方法选择、刷新方法、执行按钮。
2. 第二行放置 PHP 运行时选择与刷新 PHP 按钮。

设计原则：

1. 将高频操作横向整合。
2. 模拟参考图中“请求方法栏 + 路径栏 + 操作按钮”的工作台节奏。
3. 用更少的垂直空间承载更多关键操作。

### 5.2 中部内容工作区

中部区域由两个主要部分组成：

1. `方法信息卡片`
2. `参数编辑卡片`

其中：

1. 方法信息卡片展示当前方法签名、方法类型、参数概览、当前执行提示。
2. 参数编辑卡片承载现有参数输入组件，不改变现有服务方法与控制器方法的输入能力。

### 5.3 请求上下文区

控制器方法的请求上下文编辑区继续保留当前的 `Query / Body / Headers` 结构，但将其重组为更轻量的工作台式卡片。

保留项：

1. `Query / Body / Headers` 顶层标签结构。
2. `Body` 下的 `none / form-data / x-www-form-urlencoded / json` 模式切换。
3. 现有请求输入结果组装逻辑。

调整方向：

1. 标签页视觉从传统 Swing 风格转为更扁平的导航条。
2. `Body` 子模式更接近模式切换器而不是厚重标签页。

### 5.4 底部结果面板

底部区域继续保留现有的双态切换能力：

1. 默认显示执行结果。
2. 在编辑单个服务参数时切换到参数编辑态。

结果面板需要强化状态层次，至少覆盖：

1. 运行中
2. 成功
3. 异常
4. 无返回但执行完成

## 6. 现有组件映射

为了降低回归风险，本次重设计以“重新组织已有组件位置”为主。

组件映射如下：

1. `searchField`：进入顶部操作头部区。
2. `methodComboBox`：进入顶部主选择行。
3. `phpExecutableComboBox`：进入顶部次操作行。
4. `parameterArea`：进入方法信息卡片中的参数概览区域。
5. `argsEditorHost`：进入参数编辑卡片。
6. `requestContextPanel`：进入请求上下文卡片。
7. `resultArea / bottomPanel`：进入底部结果面板。
8. `summaryLabel + detailArea`：合并进方法信息卡片，不再以传统说明栏独立存在。

## 7. 视觉与主题规则

### 7.1 整体基调

浅色主题下保持 JetBrains 原生可读性，通过卡片分区、边距、标题层级和按钮主次提升质感。

深色主题下增强工作台气质，重点体现：

1. 更清晰的背景层次。
2. 更聚焦的主按钮。
3. 更轻的描边。
4. 更明确的选中态和状态块。

### 7.2 背景层次

整体形成三层背景关系：

1. 页面底色：使用 IDE 默认面板背景。
2. 卡片底色：与页面底色拉开轻微对比。
3. 输入区与结果区内层底色：再向内形成一层弱分组。

这样可以在深色主题下接近参考图中的工作台分层，同时在浅色主题下不显得突兀。

### 7.3 卡片样式

顶部操作区、方法信息区、参数编辑区、请求上下文区、结果区统一采用卡片容器样式。

卡片统一规则：

1. 使用圆角。
2. 使用细边框。
3. 使用 12 到 16 像素级别内边距。
4. 保证标题与正文之间的稳定留白。

### 7.4 标题与层级

每个卡片使用短标题，例如：

1. 方法信息
2. 执行参数
3. 请求上下文
4. 执行结果

主方法签名应成为页面中最醒目的信息块，承担参考图中接口标题的角色。方法类型、参数数量和 PHP 运行时状态作为次级信息展示，避免与主标题争夺视觉焦点。

### 7.5 按钮规则

按钮层级如下：

1. `执行` 为主按钮，使用强调色。
2. `刷新方法` 与 `刷新 PHP` 为次按钮，使用弱背景或边框风格。
3. 参数单项编辑确认按钮保持清晰，但不抢占主执行按钮的视觉优先级。

深色主题下主按钮可以适当借鉴参考图中的饱满高亮感，但仍应服从 IDE 当前 accent color。

### 7.6 标签与切换器

请求上下文的标签页与 `Body` 内模式切换都需要向扁平化、工作台化方向调整。

规则如下：

1. 顶层 Tabs 更像导航条。
2. 子级模式切换更像胶囊式切换器。
3. 选中态突出，未选中态弱化。
4. 深色主题下选中态更接近参考图，浅色主题下保持克制。

### 7.7 表格与输入区

请求参数表格的默认 Swing 工具感需要弱化。

优化方向：

1. 减弱网格线。
2. 适当增加行高与单元格留白。
3. 使表头弱强调、正文更清晰。
4. 统一文本输入区、JSON 输入区和滚动容器的边框与内边距风格。

### 7.8 状态反馈

执行状态需要具备清晰语义，但不依赖整块大面积着色。

状态语义至少包含：

1. 运行中：中性高亮
2. 成功：成功色
3. 失败：错误色
4. 提示：弱提示色

这些状态优先体现在方法信息区或结果区的状态标签中，以便用户快速扫读。

## 8. 主要重组类与保留项

### 8.1 主要重组类

本次主要调整以下类：

1. `MethodInvokeToolWindowPanel`
2. `ControllerRequestEditorPanel`
3. `RequestParameterTablePanel`

其中：

1. `MethodInvokeToolWindowPanel` 是主布局重组中心。
2. `ControllerRequestEditorPanel` 主要调整请求编辑区的层级与排版。
3. `RequestParameterTablePanel` 主要调整工具条、表格和滚动区视觉样式。

### 8.2 明确保留的逻辑

以下逻辑默认保留，不作为本轮改动目标：

1. 方法搜索与筛选逻辑。
2. PHP 运行时检测逻辑。
3. 最近记录恢复逻辑。
4. 执行线程模型。
5. `ServiceMethodInputPanel` 的参数采集逻辑。
6. `ControllerRequestEditorPanel.buildRequestInput()` 的请求组装逻辑。
7. `RequestParameterTableModel` 的数据模型。
8. `bottomPanel` 的结果态与参数编辑态双卡切换机制。

## 9. 新增样式支撑层

为了避免样式代码分散，本次应新增小范围 UI 样式辅助代码，用于集中处理：

1. 主题颜色读取
2. 卡片边框与内边距
3. 状态标签样式
4. 主次按钮样式
5. 文本区与滚动区样式

该支撑层应以轻量复用为目标，不做复杂主题框架。

## 10. 实现边界

本次实现边界如下：

1. 优先重做方法直调页的主工作台观感。
2. 只做最小范围的概览页视觉统一。
3. 不新增功能。
4. 不修改运行时协议和调用协议。
5. 不做复杂自定义绘制，以控制维护成本。

## 11. 验证方式

### 11.1 代码层验证

1. 原有相关测试应继续通过。
2. 如新增布局相关测试，仅保留本轮开发验证所需的最小测试代码。

### 11.2 运行层验证

建议执行：

```powershell
.\gradlew.bat test
.\gradlew.bat runIde
```

### 11.3 人工验收重点

人工验收需要重点确认：

1. 顶部操作区是否更紧凑。
2. 参数区与请求上下文区是否更清晰。
3. 深色主题下是否明显更接近参考图工作台气质。
4. 浅色主题下是否仍然保持 JetBrains 原生工具窗口风格。
5. 主链路是否能够在一屏内更顺畅地完成“选方法、填参数、执行、看结果”。

## 12. 预期结果

本次 UI 重设计完成后，预期达到以下效果：

1. 功能不变，但页面观感从传统表单页升级为调试工作台。
2. 高频操作更集中，路径更顺畅。
3. 参数区、请求上下文区、结果区层次更清晰。
4. 深浅主题下都具备一致的结构语言，且深色主题更贴近参考图。
5. 为后续统一概览页、请求调试页和诊断页提供可复用的卡片与样式基础。
